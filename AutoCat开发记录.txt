- 未分析素材仍可参与任务，但只进入高质量通用镜头池。
- 分析失败不阻止素材使用。
- 分析版本升级时后台重新分析，不在任务渲染阶段补分析。

### 性能预算与验收

- 已分析素材创建任务时视觉分析调用次数必须为 `0`。
- 10,000 个虚拟切片的 100 条语义编排不超过 `15s`。
- 语义匹配不得使批量编排耗时增加超过 `2 倍`。
- 使用人工标注样本检查主体、场景和内容作用命中率。
- 完成同步保护门禁和至少 5 条语义匹配真实成片验收。

## 第五阶段：视频类型、AI 模型选择和 DeepSeek 配置

### 修改范围

- 新增 `autokat/core/ai_providers.py`。
- 重构 `autokat/core/writer.py`，提供明确 Provider：
  - `LocalWriterProvider`
  - `DeepSeekWriterProvider`
- 禁止 DeepSeek 失败后静默切换本地模型。
- 选定 Provider 重试失败后，只允许使用明确标记的确定性安全模板，不切换其他 AI Provider。
- AI 文案提示词注入选中素材的能力摘要。
- 支持视频类型：
  - 自动判断。
  - 商品推荐。
  - 口播讲解。
  - 氛围记录。
  - 音乐卡点。
  - 随机混剪。
- 不同类型只改变编排评分、镜头节奏和转场密度，不修改字幕或配音时间轴。
- 音乐卡点复用现有 BPM 检测，但仍以 TTS 权威时长结束。

### DeepSeek 配置

- Step 2 增加 AI 模型选择，默认本地模型。
- 仅选择 DeepSeek 时显示配置和测试连接入口。
- 配置包括 API Key、API 地址、模型名称、保存和真实测试。
- API Key 保存到 macOS Keychain，不再写入 `.env` 或安装包。
- API 地址和模型名称保存到用户设置。
- 测试连接必须验证：
  - 地址可访问。
  - 密钥有效。
  - 模型存在。
  - 返回 JSON 和 `choices/message/content` 格式正确。
  - 错误信息可读。

### 数据库与 UI

- `tasks.config` 和 `scripts.tts_config` 增加 `writer_provider`、`video_type`、`intent_version`。
- 保留现有四步流程，视频类型默认“自动判断”。

### 兼容与回滚

- 旧任务默认 `video_type=random_mix`。
- 旧 `.env` Key 首次启动时迁移到 Keychain，迁移成功后停止依赖 `.env`。
- 本地与 DeepSeek Provider 独立测试，禁止跨 Provider 降级。

### 性能预算与验收

- DeepSeek 配置测试必须发起真实请求。
- 模拟 DeepSeek 失败，确认不会调用本地 Qwen。
- 自动视频类型选择和六种显式类型均生成真实成片。
- 完成同步保护门禁。

## 第六阶段：分级质量验收、自动修复和结果摘要

### 修改范围

- 新增 `autokat/core/quality.py`。
- 整合现有：
  - `tools/validate_sync.py`
  - `tools/validate_content_sync.py`
  - `autokat/core/content_sync.py`
- 所有成片执行快速验收：
  - 音频、视频、容器时长。
  - 字幕时间范围。
  - 黑屏、停帧、静音和文件完整性。
  - 最终时长与整数帧/采样点一致性。
- 深度 ASR/OCR 默认采用风险分层抽样：
  - 默认抽取批次 `10%`。
  - 最少 5 条，最多 10 条。
  - 覆盖首条、末条、随机条目、高风险条目和自动修复条目。
- 字幕、TTS 或最终合成链路发生代码修改时，开发和发布验收强制全量深度验证。
- 阻断项自动修复一次：
  - 字幕偏移：按最终 MP4 ASR 重新校准并重渲染。
  - 黑屏、停帧或动态画面不足：重新编排并重渲染。
  - 时长不一致：重新构建最终合成。
- 自动修复一次后仍失败，成片标记失败且不交付。

### 数据库与 UI

- 新增 `quality_runs`、`quality_results`。
- Step 4 和任务详情默认显示简洁摘要：
  - 已通过数量。
  - 自动修复数量。
  - 阻断失败数量。
  - 快速/深度验收状态。
- 技术报告默认折叠，展开后显示同步误差、ASR/OCR、黑屏、停帧、缓存和性能详情。

### 兼容与回滚

- 旧任务无 QA 记录时显示“未执行新版验收”，不影响播放。
- QA 失败不删除成片，保留在诊断目录，但不进入正式交付列表。
- ASR/OCR 模型按需自动下载到独立 QA 缓存，不加入正式渲染链路。

### 性能预算与验收

- 快速验收必须覆盖全部成片。
- 100 条任务默认深度验收不超过 10 条。
- 深度抽样总耗时不超过批次渲染耗时的 `20%`，并单独记录。
- 自动修复最多一次，防止无限重试。
- 完成同步保护门禁和完整 `app-test`。
